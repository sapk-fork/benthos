Performs operations against TiKV for each message with RawKV api, allowing you to store or retrieve data within message payloads.

Introduced in version 4.25.0.

```yml
# Config fields, showing default values
label: ""
tikv:
  address: [] # No default (required)
  id: ${! json("id") } # No default (required)
  content: "" # No default (optional)
  operation: get
```

When upserting, documents must have the `content` property set.

## Fields

### `address`

TiKV address.


Type: `array`  

```yml
# Examples

address: 127.0.0.1:2379
```

### `id`

Document id.
This field supports [interpolation functions](/docs/configuration/interpolation#bloblang-queries).


Type: `string`  

```yml
# Examples

id: ${! json("id") }
```

### `content`

Document content.


Type: `string`  

### `operation`

Couchbase operation to perform.


Type: `string`  
Default: `"get"`  

| Option | Summary |
|---|---|
| `delete` | delete a document. |
| `get` | fetch a document. |
| `put` | creates a new document if it does not exist, if it does exist then it updates it. |
